# pc-pc-journal

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容各自独立，不针对特定域名或具体网站，主要作为静态页面集合进行维护和展示。

## 项目简介

`pc-pc-journal` 是一个以 HTML 页面归档为主要目的的仓库。页面内容涵盖技术笔记、文档展示、实验性页面等，适合作为个人或小团队的知识沉淀与页面托管空间。所有页面均为静态 HTML，可直接通过 GitHub Pages 或其他静态托管服务访问。

## 目录说明

```
pc-pc-journal/
├── index.html          # 仓库首页 / 页面导航
├── pages/              # 存放各独立 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/             # 共用资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有独立页面存放于此，每个文件为一个完整 HTML 页面。
- `assets/`：页面共享的样式、脚本及媒体资源，避免重复引用。

## 页面归档说明

- 每个 HTML 页面均保持独立，不依赖其他页面内容。
- 页面内可引用 `assets/` 下的共用资源，但不应包含外部不可控链接。
- 归档页面主要用于展示、参考或存档，不保证实时更新或长期可用。
- 如需新增或修改页面，请遵循现有目录结构与命名规范。

## 维护说明

- 本仓库由维护者不定期更新，欢迎通过 Issues 提交建议或问题。
- 提交 Pull Request 时，请确保新增页面不包含敏感信息、恶意代码或外部依赖。
- 页面内容应保持简洁、清晰，避免冗余或过大的资源文件。
- 仓库不承诺任何形式的服务可用性或内容准确性。
